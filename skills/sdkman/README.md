# SDKMAN! Skill

Command reference and usage guide for SDKMAN!, the Software Development Kit Manager.

## Overview

This skill provides comprehensive SDKMAN! command references and best practices for managing multiple SDK versions on Unix-based systems.

## Features

- Complete command reference for all SDKMAN! operations
- Installation and version management guides
- Project-specific SDK configuration with `.sdkmanrc`
- CI/CD integration patterns
- Troubleshooting common issues
- Best practices and workflows

## Installation

1. Download `sdkman.zip` from the [releases page](https://github.com/jack482653/claude-toolkit/releases)
2. Go to Claude.ai → Settings → Capabilities → Skills
3. Upload the ZIP file

## Usage

Once installed, Claude will automatically reference this skill when you ask about:

- SDKMAN! commands and usage
- Installing or managing SDKs (Java, Groovy, Scala, Kotlin, etc.)
- Switching between SDK versions
- Configuring SDKMAN! settings
- Setting up project environments

### Example Queries

**Installation**
> "How do I install Java 21 using SDKMAN?"

**Version Management**
> "What's the difference between 'sdk use' and 'sdk default'?"

**Project Setup**
> "How do I set up automatic SDK switching for my project?"

**Troubleshooting**
> "Why isn't my SDK version switching when I change directories?"

## What's Included

- **Quick Command Reference**: Fast lookup for common commands

- **Configuration**: Settings and customization options
- **Best Practices**: Recommended workflows and patterns

## Use Cases

- **Development Environment**: Manage multiple Java versions for different projects
- **Team Collaboration**: Ensure consistent SDK versions across team members
- **CI/CD**: Automated SDK installation and configuration
- **Testing**: Quick switching between versions for compatibility testing

## Requirements

- SDKMAN! must be installed on your system
- Unix-based operating system (Linux, macOS, WSL)
